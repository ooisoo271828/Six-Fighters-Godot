Scope: 节点里程碑内须可验收的 **投射物运动/发射** 类型代号；实现结构约定；延后类型。
Related: `docs/design/visual-rules/hero-asset-pipeline-spec.md`; `docs/tech/architecture/client-rendering-and-assets.md`; `docs/design/other/product-combat-positioning.md`

## 1. 实现结构（方案 1）

- **绝大多数技能** = **运动类型 × 发射模式 × 美术 strip**（`vfx`/投射物资源由 tag 与路径约定）。
- **少数招牌** 允许单独脚本，不破坏默认管线。

## 2. 光束（类型 J）

- **光束 / 扫掠** 归入 **投射物层**：与实体飞弹 **共用调度、池化、生命周期** 思路（见 Web 客户端实现时的 `ArenaScene` / 投射物管理器）。

## 3. V1 须验收类型（里程碑内可玩点到）

**包含**：`A` `B` `C` `D` `F` `G` `H` `J` `L`
**明确延后**：`E`（抛物）、`I`（子母/延迟分裂）、`K`（贴地/沿地）

| 代号 | 含义 | 行为描述 | 工程参数要点 |
|------|------|---------|------------|
| A | 直线匀速 | 固定方向，匀速飞行 | `speed` |
| B | 变速直线 | 直线但有速度变化（加速/减速） | `speed` + `acceleration` |
| C | 弱追踪 | 飞行中有较大游荡范围，花时间才命中目标 | `speed` + `turnRate`（低，如 1~2 rad/s）|
| D | 强追踪 | 游荡范围小，追踪快而有效，快速命中 | `speed` + `turnRate`（高，如 4~6 rad/s）|
| F | 弹跳 | 碰壁/碰目标后弹开 | `bounceCount` + ` restitution` |
| G | 扇形区域（火焰之手）| 以施法者为中心，向前方扇形区域扩散的火焰/能量区域；区域内目标同时受击 | `spreadAngle` + `maxRadius` + `count` |
| H | 环形径向 | 从中心向外扩散的环形技能，类魔兽世界冰环 | `expandSpeed` + `maxRadius` |
| J | 光束/扫掠（投射物层）| 射线/扫过类技能 | `duration` + `width` |
| L | 轨道/绕体 | 沿指定轨迹持续移动的实体；环形轨道时=绕体 | `path`（直线/曲线/环形）+ `duration` |

### 3.1 C vs D 区分标准（弱追踪 vs 强追踪）

核心差异在于 **转向率（turnRate）** 和 **游荡范围**：

- **C（弱追踪）**：转向率低（1~2 rad/s），游荡范围大，命中时间长，轨迹弯曲明显
- **D（强追踪）**：转向率高（4~6 rad/s），游荡范围小，追踪紧凑有效，命中快

### 3.2 H（环形径向）澄清

- **典型代表**：魔兽世界冰环术 —— 以施法者为中心，向外扩展的环形效果
- **行为**：从中心点向外匀速/加速扩展，形成环形区域
- **区分**：不是绕体（L），而是扩散区域（H）

### 3.3 L（轨道/绕体）澄清

- **典型代表**：龙卷风（reg05）—— 沿轨迹持续移动的实体
- **行为**：技能实体沿预定路径移动；路径为环形时=绕体
- **扩展性**：v1 版本轨迹为直线；后续可扩展为曲线、环形等复杂路径

## 4a. C型（弱追踪）Ghost Fire Skull 技能规格

### 4a.1 视觉规格

| 元素 | 规格 |
|------|------|
| 核心 | 代码绘制骷髅头：深色椭圆头部（约 10px 宽）+ 两个眼眶空洞 + 下颚轮廓 |
| 外围鬼火 | 双层火焰包裹核心；外层橙红 `#ff6b35`（较大），内层青绿 `#4dff91`（较小），带随机 Flicker 效果 |
| 尾迹 | 1~1.5 个身位（约 10~14 个采样点），鬼火青绿色渐变消散；**不超过两个身位** |
| 整体配色 | 鬼火主题：青绿核心 + 橙红外焰 |

### 4a.2 飞行参数

| 参数 | D型参考值 | C型设计值 | 说明 |
|------|----------|----------|------|
| speed | 320 px/s | 280 px/s | 稍慢，留足游荡时间 |
| turnRate | 5.5 rad/s | **1.8 rad/s** | 核心区别：低转向率 |
| 漂移初始强度 | 35-70 | **70-110** | 更大散漫度 |
| 漂移衰减时间 | 0.4s | **2.2s** | 慢衰减=长时间散漫 |
| 初始朝向偏移 | ±1.8 rad | **±2.2 rad** | 更大散角 |
| 子弹数量 | 9-12 颗 | **9-12 颗** | 与 reg04 一致 |

### 4a.3 游荡行为（核心区别于D型）

每颗子弹独立运行"游荡冲动"计时器：

- **触发间隔**：每颗子弹 200-400ms 随机间隔触发一次
- **触发效果**：给子弹一个**垂直于运动方向的瞬时速度冲击**，产生甩尾/绕圈效果
- **螺旋加成**：约 30% 概率额外附加"绕圈力"，产生 1-2 圈螺旋轨迹后再继续追踪
- **随时间收敛**：>1.5s 后，游荡冲动触发概率逐渐降低，子弹逐渐回归追踪

效果：子弹们更自由散漫，有些子弹飞到空中绕圈后再命中目标，整体轨迹比强追踪（D型）弯曲夸张得多。

### 4a.4 命中精度（故意降低）

- **命中点偏移**：目标当前位置 + 随机偏移 0-40px（部分子弹打偏）
- **延迟预测**：部分子弹命中"目标 0.8 秒前所在位置"（飞行时间越长偏移越大）
- **结果**：八九不离十，但不精确

### 4a.5 AoE片伤害补偿（命中后）

- **爆炸半径**：60px
- **伤害范围**：圈内 **100% 技能伤害**，圈外无伤害（全伤害无衰减）
- **视觉**：鬼火爆发环，青绿色向外扩散消散（约 300ms）
- **补偿逻辑**：命中精度不足但杀伤范围弥补，单发覆盖面积大

### 4a.6 工程实现要点

- 游荡冲动计时器：每颗子弹独立 `wanderTimer`（200-400ms 随机）
- 漂移衰减：`currentDrift = driftStrength * Math.exp(-elapsedTime / 2.2)`
- AoE 命中：在 `spawnMissileImpactBurst` 基础上扩展，支持范围伤害圈渲染
- 尾迹采样：sampled 数组限制在 14 个点以内（约 1.5 个身位）

## 4b. F型（弹跳）Chain Lightning 闪电链 技能规格

### 4b.1 视觉规格

| 元素 | 规格 |
|------|------|
| 电流线段 | 电弧/电流束外观——锯齿状折线而非光滑直线，带发光叠加效果 |
| 车身长度 | 固定（如 120px），折射时长度不变 |
| 入射部分 | 从尾端向折射点收缩的动画 |
| 出射部分 | 从折射点向新目标延伸的动画 |
| 传导动画 | 可见的电流从旧目标传导到新目标的动画过程 |

### 4b.2 飞行参数

| 参数 | 设计值 | 说明 |
|------|--------|------|
| 折射上限 | 10 次 | 配置值，消耗完消失 |
| 搜索半径 | 200 px | 首个命中目标为中心，半径内搜索 |
| 车身总长 | 120 px | 固定，两段加起来等于总长 |
| 飞行速度 | 600 px/s | 电速，要快 |
| 入射收缩动画 | 200 ms | 尾端滑动收缩到命中点 |
| 到站停留/传导闪烁 | 60 ms | 命中点蓄力/传导闪烁特效 |
| 出射延伸动画 | 150 ms | 头端滑动延伸到下一目标 |

### 4b.3 闪电火车"车身"算法

- 尾端：跟随上一个命中目标（上一个弹射点）
- 头端：跟随当前追踪的目标
- 约束：尾端到头端的总距离固定，当头端向新目标移动时，尾端同时向折射点收缩，保持总长恒定
- 折射时：旧入射线段从尾端向折射点缩短消失；新出线段从折射点向新目标生长

### 4b.4 目标选择规则

1. 从当前目标出发，在搜索半径内搜索所有**未弹射过**的合法目标
2. 如果有未弹射目标：**随机选择**其中一个
3. 如果全部已弹射过：允许重复选择，继续消耗剩余次数
4. 次数耗光时在最后一个目标处消失
5. 极端情况：范围内只有少量目标时，会在少数目标间来回弹射直至次数耗尽

### 4b.5 伤害判定

- **仅在碰撞瞬间判定一次**，车身存在仅为视觉

### 4b.6 工程实现要点

- 用 `time.addEvent` + 手动每帧更新绘制电流线段（避免 Arc radius tween bug）
- 电流线用多段折线（锯齿）而非单条直线，模拟电流走势
- 搜索下一个目标时维护一个 `visited` Set，避免重复选择（尽量）
- 车身长度约束：头尾距离超过总长时，强制尾端跟随以保持总长
- 状态机：FLYING → INCOMING → DWELL → OUTGOING → FLESHING → ... → DONE

### 4b.7 命中特效（奥丁雷击风格，四层叠加）

| 层级 | 元素 | 规格（整体缩小40%后）|
|------|------|------------------|
| 第一层 | 核心闪光 | 白金三色圆爆发，持续120ms，半径3.6~16 |
| 第二层 | 电弧四射 | 6-9条锯齿电弧向外延伸180ms，长度12~39px |
| 第三层 | 冲击波环 | 青绿+白色双层圆环向外扩散280ms，半径2.4~31 |
| 第四层 | 电光粒子 | 10-16个小电流点向外飞散100-200ms，速度30-78px |
| 全局 | 屏幕震动 | 120ms，强度0.005 |

配色：橙金 `#ffc88f` / 青绿 `#4dff91` / 亮白 `#ffffff`

## 5c. G型（扇形区域）Burning Hands 火焰之手 技能规格

### 5c.1 视觉规格

| 元素 | 规格 |
|------|------|
| 扇形火焰体 | 从施法者位置向前方 90° 扇形区域扩散的火焰云 |
| 火焰扩散 | 从中心向外匀速/加速扩展，呈现火焰扑面而来的感觉 |
| 命中特效 | 火焰区域内所有目标同时受到火焰爆炸 |
| 火焰粒子 | 橙红色火球 + 黄色内核，带随机 Flicker |

### 5c.2 飞行参数

| 参数 | 设计值 | 说明 |
|------|--------|------|
| 扇形角度 | 90°（±45°） | 以施法者朝向来定位中心轴 |
| 扩散最大半径 | 120 px | 火焰前锋扩散到的最远距离 |
| 扩散时长 | ~350 ms | 从施法者扩散到最大半径 |
| 火焰弹数量 | 12-16 颗 | 扇形区域内随机分布的独立火球 |
| 每颗火球飞行距离 | 50-80 px | 火球从原点飞出一段后爆炸 |
| 爆炸半径 | 约 18 px | 每颗火球的 AoE 范围 |

### 5c.3 行为逻辑

1. **施法**：从施法者掌心位置向前方扇形区域同时发射多颗火焰弹
2. **扇形分布**：12-16 颗火球在 90° 扇形内随机角度+随机距离分布
3. **扩散**：火球从原点向外飞出一小段（约 50-80px）后爆炸
4. **区域伤害**：所有在爆炸半径内的目标同时受到伤害
5. **无追踪**：纯直线飞行，无转向

### 5c.4 目标选择规则

- 扇形区域内**所有**合法目标同时受到伤害
- 不限制目标数量（只要在扇形区域内+爆炸半径内）

### 5c.5 工程实现要点

- 12-16 颗火焰弹在 90° 扇形内随机初始化位置和角度
- 每颗火球独立计时，独立爆炸
- 用 `time.addEvent` + 手动每帧更新绘制
- 扇形 telegraph（蓄力提示）在扩散前先显示扇形区域

## 5d. H型（环形径向）Ice Ring 冰环 技能规格

### 5d.1 视觉规格

| 元素 | 规格 |
|------|------|
| 冰环主体 | 以施法者脚下为中心，向外扩散的白色冰环；扩散期→持续期→消退期 |
| 三层叠加 | 外层冰环亮线（多层叠加发光感）+ 霜雾底座（大范围淡蓝填充） |
| 冰晶粒子 | 24颗冰晶粒子沿圆周均匀分布，缓慢旋转闪烁，大小/相位错开 |
| 中心光晕 | 施法者脚下6个淡蓝色光晕粒子，呼吸式闪烁 |
| 整体配色 | 白色/青白/淡蓝色系，冰霜感 |

### 5d.2 时间轴

| 阶段 | 时长 | 说明 |
|------|------|------|
| 扩散期 | 0-300ms | ease-out：冰环从中心快速扩散到最大半径160px |
| 持续期 | 300-800ms | 冰环维持，呼吸脉动（半径±3%），冰晶粒子闪烁 |
| 消退期 | 800-950ms | 渐隐消失，最后微微扩大一点（膨胀感） |

### 5d.3 命中判定

- 扩散完成后每80ms在环形范围内判定一次
- 命中条件：目标距中心 ≤ 当前半径 × 1.05
- 命中反馈：中等强度（蓝色闪烁）

### 5d.4 工程实现要点

- 用 `time.addEvent` + 手动每帧更新（避免 Arc radius tween bug）
- 霜雾底座用不规则多边形（每帧随机边缘）制造有机感
- 冰晶粒子每帧独立计算位置/大小，不累积误差

## 5e. J型（光束/扫掠）Plasma Beam 等离子光束 技能规格

### 5e.1 视觉规格

| 元素 | 规格 |
|------|------|
| 光柱结构 | 三层重叠（从内到外）：内核(纯白4px) → 中层(青白14px) → 外层(蓝色48px) |
| 三层关系 | 三层以光柱中心线为轴完全重叠，仅颜色/alpha不同；无间距间隔 |
| 边缘处理 | 每层边缘独立随机不规则（每帧重算），非几何平整边界 |
| 出现过程 | 焰心渐变：内核先出现(0-200ms)→中层(200-400ms)→外层(400-580ms)，ease-out缓动 |
| 消失过程 | 外焰先散(100ms)→中焰(200ms)→内核最后(300ms)，反向渐变 |
| 命中特效 | 命中点三层闪光爆发（白→青白→蓝），120ms消失 |

### 5e.2 时间轴

| 阶段 | 时长 | 说明 |
|------|------|------|
| 内核出现 | 0-200ms | 内核从0→4px，alpha 0→0.95，ease-out |
| 中层出现 | 200-400ms | 中层从0→14px |
| 外层出现 | 400-580ms | 外层从0→48px |
| 全开维持 | 580-2080ms | 三层全开，维持1.5秒 |
| 外层消退 | 2080-2180ms | 外层先消失，100ms |
| 中层消退 | 2180-2380ms | 中层接着消失，200ms |
| 内核消退 | 2380-2680ms | 内核最后消失，300ms |

### 5e.3 命中抖动

- 命中后光束朝向命中点方向抖动（指数衰减）
- 抖动强度：命中瞬间±7px，120ms内快速衰减到0
- 三层抖动幅度不同：外层最大，内核最小

### 5e.4 工程实现要点

- 用 `time.addEvent` + 手动每帧绘制三层（不用tween）
- 每层独立 alpha/width 曲线，三层独立计算
- 命中触发在首帧即发生（光束出现即命中）
- 外层边缘随机用 `FloatBetween(-0.3, 0.3)`，内核用 `(-0.06, 0.06)`

## 5. 交付注意

- **九类同里程碑验收** 工程面较大；产品侧可选择 **尽量并行推进九类**（人力与排期按并行假设规划），或 **内部分 sprint** 实现顺序、对外仍为一个里程碑 —— 以当期排期共识为准。
- 与 **hazard telegraph（windup 片）** 的分层与错位原则见 `combat-presentation-spec.md` §3.3。
- 具体帧动画与 `vfx/<skillTag>/` 约定见 `hero-asset-pipeline-spec.md`。
