# 技能设计文档 目录说明

本目录存放所有技能的设计文档。

---

## 文件命名规范

```
{序号}_{技能类型缩写}_{技能名称}.md
```

### 序号规则
- 从 `001` 开始三位数字，按技能设计/实现顺序递增
- 删除某个技能时，序号**不回收**、**不重排**，避免引用混乱

### 技能类型缩写

| 缩写 | 含义 | 说明 |
|------|------|------|
| `proj` | Projectile 发射类 | 有飞行弹道的技能（火球术、冰箭等） |
| `aoe` | Area of Effect 范围类 | 以某点为中心的范围爆发（爆炸、闪电链） |
| `dot` | Damage Over Time 持续伤害 | 地面持续伤害区域（毒雾、火焰地带） |
| `buff` | Buff 增益类 | 对自身/盟友的增益效果 |
| `debuff` | Debuff 减益类 | 对敌人的减益效果 |
| `summon` | Summon 召唤类 | 召唤单位协助战斗 |
| `dash` | Dash 位移类 | 冲刺、闪避等位移技能 |

### 文件名示例

```
001_proj_fireball.md            # 火球术（发射类）
002_aoe_lightning_chain.md     # 闪电链（范围类）
003_dot_flame_ground.md        # 火焰地带（持续伤害）
004_buff_speed_boost.md        # 加速增益（增益类）
005_proj_ice_arrow.md          # 冰箭（发射类）
```

## 文档结构规范

每个技能设计文档必须包含以下章节（详见 `skill_design_template.md`）：

1. **元数据**：Status / Version / Owner / Last Updated / 关联文件
2. **基本信息**：技能类型、优先级、设计状态
3. **视觉设计**：按技能阶段描述视觉表现
4. **参数配置**：可程序化生成的参数表
5. **验证记录**：迭代修改历史

---

## 协作规则

1. **新增技能**：先在本文档目录创建设计文档，再开始实现
2. **修改技能**：先更新设计文档，再修改代码/资源
3. **删除技能**：将文档 Status 改为 `Deprecated`，不要删除文件
4. **跨机器协作**：文档中使用**相对路径**引用项目内资源，避免绝对路径
5. **AI 协作**：将本文档路径作为上下文提供给 AI 助手，作为技能设计的标准依据
