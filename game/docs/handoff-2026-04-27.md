## 1. 项目概况

Six-Fighters-Godot 是一个基于 Godot 4.6.2 的 2D 像素风战术战斗游戏，核心特色是 POE 风格的技能变形组合系统。项目从 Web 版（TypeScript/Phaser）迁移到 Godot。

**验证场景**：`scenes/dev/skill_demo.tscn`

## 2. 本次会话完成的核心工作

### 2.1 SkillVisualDef v2.0 增强

**文件**：`scripts/skill_system/registry/skill_visual_def.gd`  
**改动**：从 ~60 行扩展到 236 行

新增 6 个 `@export_group` 分组：
| 分组 | 前缀 | 功能 |
|------|------|------|
| Core Layers | `core_` | 多层弹体核心（主核/内核/热点/弹尖/光晕） |
| Core Jitter | `jitter_` | 核心抖动（振幅/频率） |
| Textures | `tex_` | 纹理路径（6种纹理） |
| Front Flame | `front_flame_` | 前缘火焰粒子参数 |
| Trail Particles | `trail_` | 增强拖尾粒子参数 |
| Impact Burst | `impact_` | 增强爆炸/命中参数（含震屏） |

旧参数保留向后兼容（`glow_enabled`, `explosion_*`, `core_texture_path`, `trail_texture_path`）。

### 2.2 ProjectileNode v2.0 视觉增强

**文件**：`scripts/skill_system/pools/projectile_node.gd`  
**改动**：从 ~200 行扩展到 ~500 行

**新增视觉能力**：
- **5 层 Sprite2D 核心**：光晕 → 主核 → 内核 → 热点 → 弹尖
- **前缘火焰粒子**：`_front_flame_particles` (GPUParticles2D)
- **核心抖动**：`_update_jitter()` 用 sin 函数对核心/内核/热点施加抖动
- **弹尖朝向**：`_update_nose_rotation()` 弹尖跟随飞行方向旋转
- **程序化纹理**：共享静态纹理 `_shared_circle_tex`(16x16白圆) 和 `_shared_nose_tex`(16x16右指三角)
- **屏幕震动**：`_apply_screen_shake()` 通过 Tween 实现 Camera2D 偏移
- **纹理加载兼容**：`_get_visual_str()` 优先新字段名，回退旧字段名

**节点层级**（`_setup_visuals` 创建顺序）：
```
ProjectileNode (Node2D)
├── _glow_sprite (Sprite2D)       ← 光晕层
├── _core_sprite (Sprite2D)       ← 核心层
├── _inner_sprite (Sprite2D)      ← 内核层
├── _hotspot_sprite (Sprite2D)    ← 热点层
├── _nose_sprite (Sprite2D)       ← 弹尖层
├── _trail_particles (GPUParticles2D)   ← 拖尾粒子
└── _front_flame_particles (GPUParticles2D) ← 前缘火焰
```

## 4. 已知问题和注意事项

### 4.1 重要踩坑记录

| 编号 | 问题 | 解决方案 |
|------|------|---------|
| 1 | `.tscn` 文件用 write_to_file 写入时 Tab 被换成空格 | 必须用 Godot 编辑器插件创建 |
| 2 | `.tres` 文件 [resource] 块下属性必须有 Tab 缩进 | 通过 Python 脚本写入或 Godot 编辑器 |
| 3 | GPUParticles2D 没有 direction/spread/initial_velocity_min | 这些属性在 ParticleProcessMaterial 中 |
| 4 | CompressedTexture2D 不能作为程序化纹理类型 | 用 Texture2D 代替 |
| 5 | SkillVisualDefDefault 重复声明基类属性 | 删除子类中的重复声明 |
| 6 | editor_call.py 的 executeContext.output 不是数组 | 用 print() 代替 |

### 4.2 当前遗留问题

1. **fireball_basic 视觉效果未人工验证**：代码无报错，但尚未在 skill_demo 中实际观察效果
2. **fireball_basic_vfx.tscn 是空骨架**：只有 Node2D 节点，没有实际粒子系统（当前视觉由 ProjectileNode 代码生成）
3. **旧 .tres 文件未清理**：ember_basic, ironwall_basic, moss_basic 等旧文件还在
4. **projectile_pool.gd 未更新**：可能需要适配 ProjectileNode v2.0 的新接口
5. **已删除的文件**：`scenes/arena/main.tscn`, `scenes/dev/skill_test.tscn`, `scenes/skill_system/nodes/debug_test.gd` — 需确认是否有其他引用

## 5. 下一步工作计划

### 5.1 紧急（下次会话应优先处理）

1. **验证 fireball_basic 视觉效果**
   - 在 Godot 编辑器中运行 skill_demo
   - 选择 fireball_basic，观察飞行弹体、抖动、拖尾、前缘火焰、爆炸效果
   - 根据反馈调整 SkillVisualDef 参数

2. **修复可能的视觉问题**
   - 程序化纹理（白圆/三角）可能不够美观，考虑用 assets 中的 PNG 纹理替代
   - GPUParticles2D 的 ParticleProcessMaterial 参数可能需要微调

### 5.2 短期（1-2 个会话）

3. **实现 missile_storm（导弹风暴）**
   - 需要 BEZIER_QUAD 轨迹支持（ProjectileNode 已有 _update_bezier_quad）
   - 需要多弹道机制（EmitProjectileEffect 中 projectile_count 参数）
   - 需要创建对应的 SkillDef + SkillVisualDef

4. **实现 chain_lightning（闪电链）**
   - 需要全新的链式跳跃机制（4状态机：FLYING → INCOMING → DWELL → OUTGOING）
   - 可能需要新的 Effect 类
   - 需要创建对应的 SkillDef + SkillVisualDef

### 5.3 中期（3-5 个会话）

5. **实现 AOE 类技能**（ice_cyclone, burning_hands, ice_ring）
   - 需要 AreaDamageEffect 支持
   - 可能需要新的 VisualRenderer 类型

6. **实现 plasma_beam（等离子光束）**
   - 需要光束类渲染器（三层构建 + 依次淡出）
   - 需要 beam 类型支持

7. **实现 ghost_fire_skull（幽灵火焰骷髅）**
   - 需要追踪弹机制（HOMING 轨迹）
   - 需要螺旋运动和随机转向
