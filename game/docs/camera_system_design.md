# Six Fighter GD — 镜头系统设计文档

> 本文档定义游戏的镜头架构和空间坐标系，是所有后续开发（战斗、技能、关卡、UI）的基石。
> 最后更新：2026-04-26

---

## 1. 概述

### 1.1 游戏类型
竖屏（9:16）俯视角 2D 动作游戏，固定镜头构图，支持大场景滚动。

### 1.2 核心设计理念
- **镜头即玩家操作对象**：玩家通过摇杆/方向键控制的本质是镜头中心点（锚点），而非直接控制某个角色
- **队伍软跟随**：英雄单位在以镜头锚点为圆心的范围内自由活动，不刚性绑定在镜头中央
- **竖屏构图**：始终保持 540×960 的竖屏取景，确保移动端单手操作体验一致

### 1.3 技术基础
| 参数 | 值 |
|------|-----|
| 视口尺寸 | 540 × 960 (9:16) |
| 世界空间 | 远大于视口（≥5倍），无限边界或预设大关卡边界 |
| 坐标系统 | 世界坐标，所有单位/投射物在世界空间中定位 |
| 渲染 | Camera2D 负责世界→屏幕的变换 |

## 2. 架构：锚点-跟随系统

### 2.1 三要素

```
┌───────────────────────────────────────────────┐
│                  场景世界                        │
│                                                 │
│    ┌───────┐    软跟随      ┌──────────┐       │
│    │ 英雄A  │◄──────────────┤          │       │
│    └───────┘               │  镜头锚点  │       │
│    ┌───────┐    软跟随      │ (Anchor)  │◄──────┤◄── 玩家输入
│    │ 英雄B  │◄──────────────┤          │       │
│    └───────┘               └────┬─────┘       │
│    ┌───────┐    软跟随           │              │
│    │ 英雄C  │◄──────────────     │              │
│    └───────┘                    │ 绑定跟随       │
│                           ┌────▼─────┐         │
│                           │  主镜头   │         │
│                           │ (Camera2D)│         │
│                           └──────────┘         │
└───────────────────────────────────────────────┘
```

#### 镜头锚点（CameraAnchor）
- 一个 Node2D，位于世界空间中
- 接受玩家摇杆/方向键输入，在世界空间中移动
- 移动速度固定，不受任何游戏逻辑影响
- **主镜头（Camera2D）以锚点为中心渲染**，始终保持锚点在屏幕几何中心

#### 英雄单位（Hero）
- 每个英雄拥有自身的位置和 AI 逻辑
- 目标是"跟随"镜头锚点，但有自由活动空间
- 英雄的位置 = 镜头锚点 + 阵型偏移 + AI 偏移
- AI 偏移允许英雄在战斗中出于战术原因偏离阵型位置（躲避、追击、走位等）

#### 主镜头（Camera2D）
- 每一帧将其 global_position 设置为锚点的目标位置
- 可带微量平滑跟随（smoothing），但平滑值需极小，避免操作延迟感

### 2.2 坐标系定义

```
(0,0) ────────── X (+) ───► 540
  │
  │    ┌──────────────────────┐
  │    │                      │
  Y    │    Camera Viewport   │
  (+)  │    540 × 960         │
  │    │                      │
  │    │                      │
  ▼    └──────────────────────┘
 960
```

- **世界坐标**：Godot 标准坐标系，Y 轴向下为正
- **锚点坐标**：世界坐标下的一个点，主镜头跟随此点
- **屏幕坐标**：相对于视口左上角，用于 UI 布局

> 注：所有 540/960 数值均以 Godot 引擎单位（像素）计。实际设备通过 `window/size` 缩放适配。

## 3. 镜头参数表

### 3.1 核心参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| 视口尺寸 | 540 × 960 | 竖屏固定宽高比 |
| 锚点移动速度 | 250 px/s | 摇杆控制镜头移动的速度 |
| 软跟随半径 | 350 px | 英雄围绕锚点的最大自由活动半径 |
| 硬边界阈值 | 480 px | 超过此距离触发强制拉回 |
| 英雄跟随 lerp 速度 | 3.0/s | 英雄向阵型目标位置的插值系数 |
| 英雄紧急追赶速度 | 8.0/s | 超出软半径后使用的快速插值 |
| 镜头平滑跟随 | 5.0/s | Camera2D position_smoothing_speed |
| 阵型偏移 | 见 3.2 | 每个英雄在锚点周围的固定偏移量 |

### 3.2 阵型定义

3 人小队默认阵型（锚点相对坐标）：

```
         锚点 (0, 0)
       /     |     \
  英雄A    英雄B    英雄C
 (-56,0)   (0,0)   (56,0)
```

英雄在阵型位置上叠加 AI 驱动的自由偏移，总和不得超过软跟随半径。

### 3.3 拉回机制

```
距离锚点            状态            处理方式
  0～350 px      ✅ 正常        无干预，英雄自由活动
  350～480 px    ⚠️ 超限        英雄启用紧急追赶 (lerp 8.0/s)
  >480 px        🚨 越界        直接设置英雄位置到锚点+阵型偏移
```

**紧急追赶**：
- lerp 系数从 3.0 切换为 8.0
- 英雄以"跑步"动画快速返回
- 回到 350px 半径内后 lerp 系数切回 3.0

**强制传送**：
- 瞬间将英雄 global_position 设为 `anchor_position + formation_offset`
- 触发一个闪烁/烟雾 VFX 掩盖突兀感
- 记录日志便于调试

## 4. 角色方向系统

### 4.1 方向定义

角色仅支持 **4 方向**，基于朝向向量量化：

| 角度范围 | 量化方向 | 动画后缀 |
|----------|----------|----------|
| -45°～45° | 右 (Right) | `_right` |
| 45°～135° | 下 (Down) | `_down` |
| 135°～225° | 左 (Left) | `_left` |
| 225°～315° | 上 (Up) | `_up` |

量化函数：

```gdscript
enum Direction4 { UP, DOWN, LEFT, RIGHT }

func quantize_direction4(vec: Vector2) -> Direction4:
    var angle := vec.angle()  # -PI to PI
    if angle > -PI/4 and angle <= PI/4:
        return Direction4.RIGHT
    elif angle > PI/4 and angle <= 3*PI/4:
        return Direction4.DOWN
    elif angle > -3*PI/4 and angle <= -PI/4:
        return Direction4.UP
    else:
        return Direction4.LEFT
```

### 4.2 资源紧张后备方案

如果 4 方向精灵资源无法完整提供，采用 2+2 简化方案：

- **左右**：提供完整的左右方向动画
- **上下**：在左右动画基础上做垂直偏移（不翻转），或直接使用左右动画的某个帧

所有角色的技能释放动作和受击动画遵循同样的方向规则。

### 4.3 朝向决定规则

| 状态 | 朝向决定逻辑 |
|------|-------------|
| 空闲 | 保持上一个有效朝向 |
| 行走 | 移动方向向量量化结果 |
| 施法 | 朝目标方向量化结果 |
| 受击 | 保持当前朝向，不做重定向 |

## 5. 投射物方向系统

### 5.1 与角色方向解耦

**投射物方向与角色方向完全解耦**，这是本系统的一个关键设计：

- 角色朝向：4 方向量化 ← 受限于精灵动画
- 投射物飞行：**360° 自由方向** ← 纯数学计算，无资源限制

```gdscript
# 投射物方向 = 施法者到目标的精确方向，不经过 4 方向量化
var fire_direction = (target_pos - caster_pos).normalized()
```

### 5.2 投射物轨迹类型

所有轨迹在 360° 空间内自由计算，不受角色朝向约束：

| 轨迹 | 说明 | 示例 |
|------|------|------|
| 直线 | 沿精确方向匀速飞行 | 子弹、箭矢 |
| 追踪 | 实时更新朝向目标的方向向量 | 追踪导弹 |
| 贝塞尔曲线 | 控制点偏移的弧形轨迹 | 抛射、弧线火球 |
| 正弦波 | 直线叠加垂直正弦偏移 | 闪电链、波动弹 |
| 螺旋 | 直线叠加旋转偏移 | 螺旋弹幕 |

### 5.3 视觉表现要求

- 投射物精灵应为**轴对称**或**接近圆形**，确保从任何角度看起来都自然
- 带方向性的投射物（如箭头），使用 `look_at()` 指向飞行方向
- 粒子拖尾不应假定水平/垂直方向，需在所有角度下表现正确
- 爆炸/命中特效统一为圆形辐射，与方向无关

## 6. 场景布局规范

### 6.1 关卡设计坐标参考

在竖屏构图下，场景内容的垂直分布：

```
Y=0   ┌──────────────────────┐  ──
      │                      │   ▲
      │   敌人出现区           │   │
      │   (Y: 0 ~ 350)       │  Ｃ
      │                      │  ａ
Y=350 ── ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  ｍ
      │                      │  ｅ
      │   战斗主区域           │  ｒ
      │   (Y: 200 ~ 750)     │  ａ
      │                      │  Ａ
Y=750 ── ─ ─ ─ ─ ─ ─ ─ ─ ─ ─  ｎ
      │                      │  ｃ
      │   英雄活动区           │  ｈ
      │   (Y: 600 ~ 960)     │  ｏ
      │                      │  ｒ
Y=960 └──────────────────────┘  ──
```

- 镜头锚点默认位于画面几何中心，但英雄队伍默认分布在**下半屏**
- 敌人通常在镜头上半屏出现
- 场景地图的美术内容应按此分区规则布局

### 6.2 锚点与敌人的关系

敌人在世界空间中以锚点为参考生成：

```
敌人生成区域 = 锚点位置 + (0, -250～-100)
              ← 偏移 ±200 横向散布
```

英雄阵型则保持在锚点附近：

```
英雄目标位置 = 锚点位置 + 阵型偏移 + (0, +150～+200)
```

这样设计确保竖屏构图中"敌在上、我在下"的视觉格局始终成立。

## 7. 技能演示器方案

基于上述镜头方案，技能演示器的改造设计如下。

### 7.1 场景构成

```
┌─ 技能演示场景 ───────────────────────────┐
│  ┌──────────────────────────────────────┐ │
│  │    带地砖网格的背景地面（俯视）          │ │
│  │                                      │ │
│  │         🎯 目标单位(可点击放置)        │ │
│  │               ← 火球轨迹 →            │ │
│  │         🧙 施法单位(固定位置)          │ │
│  │                                      │ │
│  │  ┌─ 左下 UI 面板 ─────────────────┐  │ │
│  │  │ 技能下拉 | 施放按钮 | 状态     │  │ │
│  │  │ 技能数: N | 命中数: M         │  │ │
│  │  └───────────────────────────────┘  │ │
│  └──────────────────────────────────────┘ │
└────────────────────────────────────────────┘
```

### 7.2 与游戏一致的镜头系统

- 演示器使用**完全相同的 CameraAnchor + Camera2D 系统**
- 施法单位固定在锚点 + 阵型偏移位置（即英雄在游戏中的常规位置）
- 目标单位可拖拽/点击放置在世界空间的任意位置
- 镜头可操作移动（用 WASD/摇杆），方便从不同位置观察技能演示

### 7.3 单位设计

**施法单位**：
- 一个用简单几何图形/占位精灵表示的 Hero 形象
- 带有 4 方向朝向指示器（一个箭头或光环标记）
- 固定在屏幕下半区（英雄活动区）
- 支持通过 UI 切换不同的英雄技能

**目标单位**：
- 简单靶标形象（箭靶/训练假人）
- 支持多个目标（通过点击不同位置添加）
- 显示血条或生命指示（可选）
- 支持右键移除或"全部清除"
- 目标可以设置不同的站位模式（散开、聚拢、一字排开），方便测试 AOE

### 7.4 地面背景

- 深色俯视地砖网格，提供空间参考系
- 可选多种背景主题（训练场、草地、岩地）
- 标出锚点中心十字线、软跟随半径（350px 半透明圈）**（仅调试模式显示）**

### 7.5 操作流程

1. 启动场景 → 进入演示模式
2. 右下 UI 选择技能
3. （可选）点击地面放置目标单位
4. 点击"施放"或按快捷键 → 技能从施法单位飞向目标
5. 镜头可自由移动（WASD/摇杆），从任意角度观察
6. 投射物命中目标 → 显示命中反馈
7. 场景中的单位和投射物不影响实际的游戏存档

## 9. 附录：边缘情况处理

### 9.1 场景边界
如果关卡有物理边界（场景墙），镜头锚点不应越过边界。当锚点被边界阻挡时：
- 镜头贴边，适当露出暗边或由关卡美术内容填补
- 超出场景部分的视口内容留黑或显示边界装饰

### 9.2 英雄全部死亡
所有英雄死亡后，镜头锚点：
- 停止响应玩家输入
- 保持死亡时的最后位置
- 等待复活/重开逻辑

### 9.3 BOSS 战特殊处理
BOSS 战场景可能比普通关卡大或小。BOSS 战时可临时调整：
- 软跟随半径可缩小（迫使队伍更紧密）
- 锚点移动速度可降低（BOSS 战更注重走位精度）

### 9.4 多人/多英雄同步
所有英雄的坐标都以世界空间为基准，不依赖屏幕位置。因此多英雄在锚点周围的分布完全由各自的 AI 偏移 + 阵型偏移叠加决定，天然支持异步行为。
