## 2. 本次会话完成的核心工作

### 2.1 导弹风暴视觉效果修复与优化

**修复：拖尾 Line2D 坐标系错误**
- **根因**：`_update_comet_trail()` 用 `global_position`（世界坐标）写入 `Line2D.points`，但 Line2D 期望的是**局部坐标**。Line2D 是 ProjectileNode 的子节点，跟随子弹移动，世界坐标赋值导致拖尾渲染偏移数百像素。
- **修复**：改用局部偏移量 `_trail_offsets`，每帧计算子弹移动量并反向补偿已有 trail 点，保证拖尾留在世界空间的正确位置。

**优化项（用户确认满意）：**

| 优化 | 改动 |
|------|------|
| 子弹头放大 + 辉光射线 | core 12→16px, glow_radius 24→14, 新增外层辉光(glow2) + 辐射射线纹理 |
| 拖尾宽度渐变 | 新增 `width_curve`，从前到后非均匀变细（带波动） |
| 弹道差异化 | Bezier 控制点前推比例随机(35%~75%)、垂直偏移随机(20~160px)、速度 ±20% |
| 错峰发射 | 在 0.05~0.8s 窗口内随机延迟，子弹陆续射出 |
| 命中爆炸（sprite 版代替 particles） | 完全重写 `_spawn_explosion()` 为 Sprite2D+Tween 粒子爆发 |

### 2.2 命中爆炸方向 Bug 排查与修复

这是今天**耗时最长**的排查过程。坑很深，涉及多层缓存和 VFX 系统架构问题：

1. **第一层 Bug**：GPUParticles2D `direction = Vector3(0,0,0)` → GPU 归一化未定义 → 默认回退 `(1,0,0)` → 全部向右喷射
2. **第二层 Bug**：`_spawn_explosion()` 调试打印不出现，因为游戏运行时 `print()` 不经过编辑器插件日志
3. **第三层 Bug**：即使修正了 `_spawn_explosion()` 的代码，**旧行为依然存在**。排查发现 `SkillVFXManager` 中的 `CPUParticles2D` 才是"向右下喷射"的元凶——它有独立的 `_on_skill_hit()` 监听同一命中事件，且 CPUParticles2D 的默认值是 `direction=(1,0)` + `gravity=(0,10)`

**最终修复**：
- 用 Sprite2D+Tween 替代 GPUParticles2D/CPUParticles2D，完全可控
- 重构 VFXManager 为层级池系统（见 2.3）
- 爆炸粒子：`spread=360°`、非零方向向量、零重力

## 5. 已知问题和注意事项

### 5.1 踩坑记录（新）

| 编号 | 问题 | 根因 | 解决方案 |
|------|------|------|---------|
| 1 | 拖尾线离子弹数百像素 | `global_position` 给 `Line2D.points`（需要局部坐标） | 改用局部偏移量 + 帧间移动补偿 |
| 2 | 粒子全部向右喷射 | GPUParticles2D `direction=Vector3(0,0,0)` GPU 归一化退化为 `(1,0,0)` | 用非零方向向量 + spread=360° |
| 3 | 受击粒子向右下喷射 | CPUParticles2D 默认 direction=(1,0), gravity=(0,10) | 重构 VFXManager，移除硬编码粒子 |
| 4 | 两套爆炸同时播放 | ProjectileNode._spawn_explosion + VFXManager 独立执行 | skill_id 白名单跳过 |
| 5 | 调试 print 不出现在日志 | 游戏运行时 print 不经过编辑器插件日志 | 在 Godot Output 面板查看 |
| 6 | 新加 class_name 不可用 | 编辑器脚本类缓存未更新 | 触发 `ei.get_resource_filesystem().scan()` |
| 7 | 新字段缩进导致解析崩溃 | 0 tab 区域写入 1 tab 字段，混合缩进 | 所有类成员缩进必须一致 |
| 8 | sed `\t` 插入字面字母 t | Git Bash sed 不认 `\t` | 用 Python 脚本替代 sed |
| 9 | 脚本修改后不生效 | Godot 资源缓存 | 删除 .godot/*.cfg 后重启编辑器 |
| 10 | `.tres` 属性值加载不一致 | Resource 缓存：某些属性返回旧值 | 硬编码参数到脚本中绕过缓存 |

### 5.2 当前遗留问题

1. **VFX 层级池效果库**：目前只预置了 7 个效果，需要根据后续技能需求扩充
2. **sprite_burst 和 ring 类型的 VFXExecutor**：预留但未实现
3. **对象池优化**：Sprite2D 爆发目前每次创建新实例，大量并发时可能有性能开销
4. **fireball_basic 的 projectile_node 专属爆炸**：目前走层级池效果，无专用 `_spawn_explosion`

## 7. 关键代码入口

| 需求 | 入口文件 | 关键方法 |
|------|---------|---------|
