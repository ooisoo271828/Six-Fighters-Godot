### 1.3 关键设计决策

**镜头系统（CameraAnchor）**：
- 玩家通过摇杆/方向键控制**镜头锚点（CameraAnchor）**，而非直接控制角色
- Camera2D 作为 CameraAnchor 的子节点，跟随锚点移动（position_smoothing = 5.0）
- 英雄在锚点周围 **350px 半径**内自由活动（软跟随），超出后用快速 lerp（8.0/s）追赶
- 超过 480px 硬边界则直接传送回锚点位置
- 英雄朝向：**4 方向量化**（上/下/左/右），带 2+2 后备方案
- 投射物方向：**360° 自由计算**，与角色方向解耦
- CameraAnchor 脚本位于 `scripts/dev/camera_anchor.gd`，可被 Arena 场景复用

**SkillDemo 演示器**：
- 与游戏内一致的镜头系统（CameraAnchor + Camera2D）
- 施法单位固定在 `anchor_pos + (0, +170)`（英雄阵型位置）
- 靶标以锚点为基准向上偏移排列（Y: -200～-320 范围）
- 场景布局遵循 "敌在上、我在下" 的竖屏构图规则
- 背景为俯视深色网格地砖 + 中心十字参考 + 350px 软跟随半径参考圈

**演示器 UI（底部半透明面板）**：
| 控件 | 功能 |
|------|------|
| 技能下拉列表 | 从 SkillRegistry 动态加载所有技能 |
| ▶ ⏸ ⏹ | 播放 / 暂停 / 停止 |
| 速度倍率按钮 | 0.5× / 1× / 2× 循环切换 |
| 受体模式按钮 | 单受体 / 双目标 / 三角阵 / 散开群 / 一字排 |
| 🔁 循环复选框 | 施放完成后自动重放 |
| 状态标签 | 显示当前状态和信息 |

**速度/暂停实现**：
- 使用 `Engine.time_scale` 控制播放速度（暂停时 time_scale=0）
- 暂停时投射物冻结，但 UI 按钮仍然响应
- 快捷键：空格 = 播放/停止，R = 切换循环
- `_exit_tree()` 中自动恢复 time_scale = 1.0

## 6. 关键设计决策备忘

1. **镜头即玩家操作对象**：玩家控制的本质是镜头锚点，而非直接控制单个角色
2. **角色方向与投射物方向解耦**：角色 4 方向（精灵限制），投射物 360°（数学自由）
3. **英雄软跟随**：锚点 → 阵型偏移 → AI 偏移，三层叠加决定英雄位置
4. **竖屏构图始终成立**：Y 轴规则（敌上我下）在任何锚点位置都不变
5. **演示器即验证工具**：演示器使用与游戏完全一致的镜头系统，所见即所得
6. **Engine.time_scale 控制播放**：速度倍率与暂停均通过 time_scale 实现
7. **右键点击无功能**：演示器当前未实现右键清除靶标功能，后续可按需添加
