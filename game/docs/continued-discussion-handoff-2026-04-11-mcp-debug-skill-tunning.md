## 1) MCP 配置调试

### 问题
- `duckduckgo-mcp` 在中国大陆网络环境无法访问（DuckDuckGo 被封锁）
- 内置 `WebSearch` 同样失败（底层调用 DuckDuckGo）

### 解决方案
安装并配置 `one-search-mcp`（local 模式）：
```json
{
  "command": "npx",
  "args": ["-y", "one-search-mcp"],
  "env": { "SEARCH_PROVIDER": "local" }
}
```
- 使用本地 Edge/Chrome 浏览器进行搜索
- 支持 `one_search`（搜索）和 `one_scrape`（抓取网页）
- 无需 API Key，免费，中国可访问

### 失效工具清单（不要浪费时间）
- `duckduckgo-mcp` — 网络封锁
- `WebSearch`（Claude 内置）— 网络封锁
- `Tavily API` — 未测试，可能也受限

### 可用工具
- `one-search-mcp` 的 `one_search` — ✅ 可用
- `WebFetch`（Claude 内置）— ✅ 可用（抓取海外大站可能被 Cloudflare 拦截）

## 3) 投射物类型定义澄清（重要）

### 类型定义（2026-04-11 澄清版）

| 代号 | 含义 | 关键行为 |
|------|------|---------|
| A | 直线匀速 | 固定方向，匀速飞行 |
| B | 变速直线 | 直线但有速度变化（加速/减速） |
| C | 弱追踪 | 飞行中有较大游荡范围，命中较慢 |
| D | 强追踪 | 游荡范围小，追踪快而有效 |
| F | 弹跳 | 碰壁/碰目标后弹开 |
| G | 扇形散射 | 多枚弹体从扇形区域同时发射 |
| H | 环形径向 | 从中心向外扩散的环形（类魔兽世界冰环） |
| J | 光束/扫掠 | 射线/扫过类技能 |
| L | 轨道/绕体 | 沿轨迹持续移动的实体；环形轨道=绕体 |

### C vs D 区分标准（弱追踪 vs 强追踪）
- **游荡范围大、花时间命中** → **弱追踪 (C)**
- **游荡范围小、快速命中** → **强追踪 (D)**
- 核心差异：追踪转向率（turnRate）高低

## 4) reg04 导弹风暴重构（今日完成）

### 问题
原版 `missile_storm` 使用预计算 Bezier 曲线，不追踪目标，属于"假追踪"。

### 重构方案
实现真正的 D 型强追踪：
- 每帧重新计算朝向目标的角度
- 高转向率（5.5 弧度/秒）限制转弯速度
- 速度 320 像素/秒
- 命中检测：距离 < 20px 时触发

### 技术实现
- 用 `tween` 作为时钟源
- 每帧用 `this.time.now` 计算真实 dt
- 追踪算法：计算目标角度与当前角度差，限制每帧最大转向角

### 修改位置
- `spawnStormMissile` 方法（行346-459）
- 保留了原有的彗星尾迹视觉（多层渐变、闪光点）
